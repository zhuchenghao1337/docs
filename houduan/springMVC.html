      # SpringMVC是什么

      基于MVC开发模式的框架，用来优化控制器。

      # 什么是MVC?

      一种开发模式，模型视图控制器，所有web应用基于MVC开发。

      M：模型层，包含实体类，业务逻辑层，数据访问层

      V：视图层，html、JavaScript、vue等，用来显示数据

      C：控制器，用来接收客户端的请求，并返回响应到客户端的一个组件，Servlet就是一个组件

      # SSM

      Spring，SpringMVC层，Mybatis

      # SpringMVC的优点

      1. 轻量级，基于MVC的框架
      2. 易于上手，功能强大
      3. 具备IOC和AOP
      4. 完全基于注解开发

      # SpringMVC开发步骤

      1. 新建项目，选择webapp模板
      2. 补全目录
      3. 修改pom.xml文件，添加资源指定，添加SpringMVC、Servlet等等依赖
      4. 添加springmvc.xml文件，指定包扫描，添加视图解析器
      5. 删除web.xml文件，新建web.xml
      6. 在web.xml文件中注册SpringMVC框架
      7. 新建index.jsp页面，发送请求给服务器
      8. 开发控制器(servlet)
      9. 添加tomcat，进行测试功能

      # @RequestMapping注解详解

      • 加在方法上，为该方法注册一个可以访问的路径

      • 加载类上，相当于包名

      • 此注解可区分get和post请求

        @Controller
public class RequestAction {
    @RequestMapping(value = "/request", method = RequestMethod.GET)
    public String requestdemo1() {
        System.out.println("我处理了get请求");
        return "main";
    }
    @RequestMapping(value = "/request", method = RequestMethod.POST)
    public String requestdemo2() {
        System.out.println("我处理了post请求");
        return "main";
    }
}

      # tomcat9解决中文编码

      原理是过滤器

      xml文件中配置
<!--中文编码过滤器-->
    <filter>
        <filter-name>encode</filter-name>
        <filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class>
        <!--参数配置-->
        <!--
        private String encoding;
        private boolean forceRequestEncoding;
        private boolean forceResponseEncoding;
        -->
        <init-param>
            <param-name>encoding</param-name>
            <param-value>UTF-8</param-value>
        </init-param>
        <init-param>
            <param-name>forceRequestEncoding</param-name>
            <param-value>true</param-value>
        </init-param>
        <init-param>
            <param-name>forceResponseEncoding</param-name>
            <param-value>true</param-value>
        </init-param>
    </filter>
    <filter-mapping>
        <filter-name>encode</filter-name>
        <url-pattern>/*</url-pattern>
    </filter-mapping>

      # action方法的返回值

      1. String:客户端资源的地址，自动拼接前缀和后缀，可以指定返回的路径 ------页面跳转
      2. Object:返回json格式的对象，自动将对象或集合转为json，必须添加Jackson依赖，一般用于ajax请求
      3. void:无返回值，一般用于ajax请求请求
      4. 基本数据类型:用于ajax请求
      5. ModeAndView:返回数据和视图对象

      # 页面跳转-请求转发和重定向

      请求转发是基于服务器的跳转

      重定向是基于客户端的跳转

      转发和重定向的本质区别？

      • 转发：是由WEB服务器来控制的。A资源跳转到B资源，这个跳转动作是Tomcat服务器内部完成的。
      • 重定向：是浏览器完成的。具体跳转到哪个资源，是浏览器说了算。
