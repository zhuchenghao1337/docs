                      # 1.什么是Redis？

                      非关系数据库，key-value模式存数据。

                      # 2.适用场景

                      1.取最新的数据

                      2.热搜

                      3.登录信息过期

                      4.分布式下新增id，全局唯一

                      redis 单线程

                      5.网站的用户分布在哪几个省 set排重

                      # 3.特点

                      1.高效

                      因为放到内存

                      2.原子性

                      主逻辑线程是单线程

                      3.支持多种数据结构

                      string

                      list a--> ["hello", "2"] 消息队列

                      hash 就是map 购物车

                      set 去重

                      zset 热搜 排行榜

                      4.稳定性 定期将数据存到硬盘（持久化）

                      5.其他：支持过期时间（登录状态过期），支持事务，消息订阅

                      # 8.高级操作

                      # 1.数据删除策略

                      数据存放在内存，内存中的数据可以通过TTL指令获取其状态
ttl返回值：正数，-1：,-2
正数：还能存活的时间
-1：永久有效
-2：已经过期的数据 或被删除的数据 或 未定义的数据
定时删除:过期就删除
惰性删除:到达过期时间，不做处理
		下次访问该数据 : 
		1.如果未过期，返回数据
		2.发现已过期，删除，返回不存在
定期删除:周期性抽查存储空间（随机抽查，重点抽查）
